Google Flow Image & Video Generation Proxy API

Introduction

The Flow API provides a unified interface for Google Flow's image and video generation capabilities. It supports text-to-image (T2I), image-to-image (I2I), text-to-video (T2V), image-to-video (I2V), first/last frame video (I2V-FL), reference-to-video (R2V), and upscaling for both images and videos.

All endpoints accept and return JSON. All endpoints use the POST method.

Base URL

http://localhost:9339

The port is configurable. All endpoint paths below are relative to this base URL. The API prefix is /api/flow.

Authentication

Each request must include a valid Google OAuth accessToken and projectId in the request body. These are used to authenticate with Google Flow's backend services.

Required Fields (every request)

Field Type Description
accessToken string Google OAuth access token
projectId string Google Cloud project ID

Error Handling

When a request fails, the response will include success: false and an error field with a descriptive message.

// Error response format
{
  "success": false,
  "error": "Invalid access token or token expired"
}
HTTP Code Meaning
200 Success (check success field)
400 Bad request / missing required fields
401 Invalid or expired access token
500 Internal server error / upstream failure
POST /api/flow/images/generate

Generate an image from a text prompt (T2I) or transform an existing image (I2I). For I2I, include a reference image as base64.

Request Body

Field Type Description
accessToken string required — Google OAuth access token
projectId string required — Google project ID
promptText string required — Text prompt describing the desired image
aspectRatio string optional — Default: IMAGE_ASPECT_RATIO_SQUARE
imageModel string optional — Default: GEM_PIX_2
seed int optional — Random seed for reproducibility
referenceImageBase64 string optional — Base64-encoded reference image for I2I mode
referenceImageMimeType string optional — Default: image/jpeg
referenceImageFileName string optional — Filename for the reference image

Aspect Ratio Values

IMAGE_ASPECT_RATIO_SQUARE IMAGE_ASPECT_RATIO_LANDSCAPE IMAGE_ASPECT_RATIO_PORTRAIT IMAGE_ASPECT_RATIO_LANDSCAPE_FOUR_THREE IMAGE_ASPECT_RATIO_PORTRAIT_THREE_FOUR

Image Model Values

GEM_PIX_2 NARWHAL IMAGEN_3_5

Response

{
  "success": true,
  "batchId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "media": [
    {
      "mediaName": "f1e2d3c4-b5a6-7890-abcd-ef1234567890",
      "mediaId": "f1e2d3c4-b5a6-7890-abcd-ef1234567890",
      "fifeUrl": "https://flow-content.google/image/...",
      "seed": 12345,
      "prompt": "A beautiful mountain landscape at sunset",
      "aspectRatio": "IMAGE_ASPECT_RATIO_SQUARE",
      "dimensions": { "width": 1024, "height": 1024 }
    }
  ],
  "remainingCredits": 95
}
POST /api/flow/images/upscale

Upscale a previously generated image to 2K or 4K resolution. Use the mediaName from the generate response as the mediaId.

Request Body

Field Type Description
accessToken string required — Google OAuth access token
projectId string required — Google project ID
mediaId string required — The mediaName from the generate response
targetResolution string optional — Default: UPSAMPLE_IMAGE_RESOLUTION_4K

Target Resolution Values

UPSAMPLE_IMAGE_RESOLUTION_2K UPSAMPLE_IMAGE_RESOLUTION_4K

Response

{
  "success": true,
  "encodedImage": "/9j/4AAQSkZJRgABAQAAAQABAAD/2wBDAAg..."
}

The encodedImage field contains the full base64-encoded upscaled image. Decode it to get the image binary.

POST /api/flow/videos/generate

Generate a video. Supports four modes depending on which image fields are provided:

T2V Text-to-Video

Only promptText, no images

I2V Image-to-Video

promptText + startImageBase64

I2V-FL First/Last Frame

promptText + startImageBase64 + endImageBase64

R2V Reference-to-Video

promptText + referenceImages[] (1-3 images)

Request Body

Field Type Description
accessToken string required — Google OAuth access token
projectId string required — Google project ID
promptText string required — Text prompt describing the desired video
aspectRatio string optional — Default: VIDEO_ASPECT_RATIO_LANDSCAPE
videoModelKey string optional — Default: veo_3_1_t2v_fast_4s — See Model Keys
seed int optional — Random seed for reproducibility
I2V / I2V-FL Fields
startImageBase64 string optional — Base64-encoded start frame image (I2V / I2V-FL)
startImageMimeType string optional — Default: image/jpeg
startImageAspectRatio string optional — Default: IMAGE_ASPECT_RATIO_LANDSCAPE
endImageBase64 string optional — Base64-encoded end frame image (I2V-FL only)
endImageMimeType string optional — Default: image/jpeg
endImageAspectRatio string optional — Default: IMAGE_ASPECT_RATIO_LANDSCAPE
R2V Fields
referenceImages array optional — 1-3 reference images for R2V mode. Each object: { base64, mimeType, aspectRatio }
referenceAudio string optional — Default: zephyr — Audio track for R2V

Video Aspect Ratio Values

VIDEO_ASPECT_RATIO_LANDSCAPE VIDEO_ASPECT_RATIO_PORTRAIT VIDEO_ASPECT_RATIO_SQUARE

Response

{
  "success": true,
  "batchId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "operations": [
    {
      "operationName": "op-uuid-1234",
      "sceneId": "scene-uuid-5678"
    }
  ],
  "remainingCredits": 95
}

Video generation is asynchronous. Use the operationName and sceneId from the response to poll for completion via POST /api/flow/videos/status.

POST /api/flow/videos/upscale

Upscale a previously generated video to 1080p or 4K resolution. Use the operationName from the generate response as the videoMediaId.

Request Body

Field Type Description
accessToken string required — Google OAuth access token
projectId string required — Google project ID
videoMediaId string required — The operationName from the generate response
resolution string optional — Default: VIDEO_RESOLUTION_1080P
aspectRatio string optional — Default: VIDEO_ASPECT_RATIO_LANDSCAPE
videoModelKey string optional — Default: veo_3_1_upsampler_1080p
seed int optional — Random seed for reproducibility

Resolution Values

VIDEO_RESOLUTION_1080P VIDEO_RESOLUTION_4K

Upscaler Model Keys

veo_3_1_upsampler_1080p veo_3_1_upsampler_4k

Response

Same structure as videos/generate:

{
  "success": true,
  "batchId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "operations": [
    {
      "operationName": "op-uuid-1234",
      "sceneId": "scene-uuid-5678"
    }
  ],
  "remainingCredits": 95
}

Upscaling is also asynchronous. Poll POST /api/flow/videos/status with the returned operationName to get the upscaled video URL.

POST /api/flow/videos/status

Poll the status of one or more video generation or upscale operations. Call this endpoint repeatedly until completed is true.

Request Body

Field Type Description
accessToken string required — Google OAuth access token
operations array required — Array of operation objects to poll

Operations Array Item

Field Type Description
operationName string required — Operation ID from generate/upscale response
sceneId string optional — Scene ID from generate response (default: empty string)

Response (Successful)

{
  "success": true,
  "completed": true,
  "operations": [
    {
      "status": "MEDIA_GENERATION_STATUS_SUCCESSFUL",
      "operationName": "op-uuid-1234",
      "sceneId": "scene-uuid-5678",
      "video": {
        "fifeUrl": "https://flow-content.google/video/...",
        "servingBaseUri": "...",
        "mediaId": "media-uuid-9012",
        "seed": 12345,
        "prompt": "A cinematic drone shot over mountains",
        "model": "veo_3_1_t2v_fast_4s",
        "aspectRatio": "VIDEO_ASPECT_RATIO_LANDSCAPE"
      }
    }
  ],
  "remainingCredits": 95
}

Response (Still Processing)

{
  "success": true,
  "completed": false,
  "operations": [
    {
      "status": "MEDIA_GENERATION_STATUS_ACTIVE",
      "operationName": "op-uuid-1234",
      "sceneId": "scene-uuid-5678",
      "video": null
    }
  ],
  "remainingCredits": 95
}

Status Values

Status Meaning
MEDIA_GENERATION_STATUS_PENDING Queued, not yet started
MEDIA_GENERATION_STATUS_ACTIVE Currently generating
MEDIA_GENERATION_STATUS_SUCCESSFUL Complete — video URL available
MEDIA_GENERATION_STATUS_FAILED Generation failed

Recommended polling interval: every 5-10 seconds. Video generation typically takes 30-120 seconds depending on model and duration.

Try It (curl Examples)

Copy and paste these examples to test the API. Replace YOUR_ACCESS_TOKEN and YOUR_PROJECT_ID with your actual credentials.

1. Generate an Image (T2I)

# Text-to-Image generation
curl -X POST http://localhost:9339/api/flow/images/generate \
  -H "Content-Type: application/json" \
  -d '{
    "accessToken": "YOUR_ACCESS_TOKEN",
    "projectId": "YOUR_PROJECT_ID",
    "promptText": "A serene Japanese garden with cherry blossoms",
    "aspectRatio": "IMAGE_ASPECT_RATIO_LANDSCAPE",
    "imageModel": "GEM_PIX_2"
  }'

2. Upscale an Image to 4K

# Upscale using mediaName from generate response
curl -X POST http://localhost:9339/api/flow/images/upscale \
  -H "Content-Type: application/json" \
  -d '{
    "accessToken": "YOUR_ACCESS_TOKEN",
    "projectId": "YOUR_PROJECT_ID",
    "mediaId": "MEDIA_NAME_FROM_GENERATE",
    "targetResolution": "UPSAMPLE_IMAGE_RESOLUTION_4K"
  }'

3. Generate a Video (T2V)

# Text-to-Video generation
curl -X POST http://localhost:9339/api/flow/videos/generate \
  -H "Content-Type: application/json" \
  -d '{
    "accessToken": "YOUR_ACCESS_TOKEN",
    "projectId": "YOUR_PROJECT_ID",
    "promptText": "A cinematic drone shot flying over misty mountains at sunrise",
    "aspectRatio": "VIDEO_ASPECT_RATIO_LANDSCAPE",
    "videoModelKey": "veo_3_1_t2v_fast_4s"
  }'

4. Generate a Video from Image (I2V)

# Image-to-Video: provide a start frame
curl -X POST http://localhost:9339/api/flow/videos/generate \
  -H "Content-Type: application/json" \
  -d '{
    "accessToken": "YOUR_ACCESS_TOKEN",
    "projectId": "YOUR_PROJECT_ID",
    "promptText": "The camera slowly zooms in as petals fall",
    "videoModelKey": "veo_3_1_i2v_s_fast_6s",
    "startImageBase64": "BASE64_ENCODED_IMAGE_DATA",
    "startImageMimeType": "image/jpeg"
  }'

5. Upscale a Video to 4K

# Upscale using operationName from generate response
curl -X POST http://localhost:9339/api/flow/videos/upscale \
  -H "Content-Type: application/json" \
  -d '{
    "accessToken": "YOUR_ACCESS_TOKEN",
    "projectId": "YOUR_PROJECT_ID",
    "videoMediaId": "OPERATION_NAME_FROM_GENERATE",
    "resolution": "VIDEO_RESOLUTION_4K",
    "videoModelKey": "veo_3_1_upsampler_4k"
  }'

6. Poll Video Status

# Poll until completed is true
curl -X POST http://localhost:9339/api/flow/videos/status \
  -H "Content-Type: application/json" \
  -d '{
    "accessToken": "YOUR_ACCESS_TOKEN",
    "operations": [
      {
        "operationName": "OPERATION_NAME_FROM_GENERATE",
        "sceneId": "SCENE_ID_FROM_GENERATE"
      }
    ]
  }'

7. Full Workflow (Generate + Poll)

#!/bin/bash
# Full workflow: generate video, then poll until done

TOKEN="YOUR_ACCESS_TOKEN"
PROJECT="YOUR_PROJECT_ID"
BASE="http://localhost:9339"

# Step 1: Generate
RESULT=$(curl -s -X POST $BASE/api/flow/videos/generate \
  -H "Content-Type: application/json" \
  -d "{
    \"accessToken\": \"$TOKEN\",
    \"projectId\": \"$PROJECT\",
    \"promptText\": \"A cat playing piano\",
    \"videoModelKey\": \"veo_3_1_t2v_fast_4s\"
  }")

echo "Generate response: $RESULT"

# Extract operationName and sceneId (requires jq)
OP_NAME=$(echo $RESULT | jq -r '.operations[0].operationName')
SCENE_ID=$(echo $RESULT | jq -r '.operations[0].sceneId')

# Step 2: Poll until done
while true; do
  sleep 10
  STATUS=$(curl -s -X POST $BASE/api/flow/videos/status \
    -H "Content-Type: application/json" \
    -d "{
      \"accessToken\": \"$TOKEN\",
      \"operations\": [{
        \"operationName\": \"$OP_NAME\",
        \"sceneId\": \"$SCENE_ID\"
      }]
    }")

  COMPLETED=$(echo $STATUS | jq -r '.completed')
  echo "Polling... completed=$COMPLETED"

  if [ "$COMPLETED" = "true" ]; then
    echo "Done!"
    echo $STATUS | jq '.operations[0].video.fifeUrl'
    break
  fi
done
